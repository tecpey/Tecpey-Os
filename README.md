## What is TecPey?

TecPey (`tecpey.ir`) is a Persian-language crypto trading platform and education ecosystem built for the Iranian market and the global Persian-speaking community. It was created with one core belief:

> **Entering digital financial markets should be more informed, safer, and more responsible.**

TecPey combines a live crypto exchange (`my.tecpey.ir`) with a free 7-term Academy, an AI Mentor, a Trading Arena simulator, and a community career system — all in one product.

تک‌پی (`tecpey.ir`) یک پلتفرم ارز دیجیتال و اکوسیستم آموزشی به زبان فارسی است که برای بازار ایران و جامعه فارسی‌زبان جهانی ساخته شده. این پلتفرم بر پایه یک باور بنیادین بنا شده:

## Architecture Overview

```
tecpey.ir (main platform)
├── / (fa-IR, RTL) — Persian landing, markets, academy, community
├── /en (en-US, LTR) — English mirror with full parity
├── /academy/** — 7-term learning path, AI mentor, simulator
│   ├── /academy/[slug] — individual lesson pages
│   ├── /academy/certificates — verified certificate dashboard
│   ├── /academy/ai-guide — AI Mentor chat
│   ├── /academy/simulator — Trading Arena
│   └── /academy/profile — student dashboard + Trading DNA
├── /markets — Live price board and swap
├── /crypto/[symbol] — Individual crypto pages
├── /coins/** — Crypto dossiers
├── /verify/[certificateId] — Public certificate verification
└── /api/** — Edge-ready API routes (CSRF-protected)

my.tecpey.ir (exchange)
└── Separate trading application (live markets)
```

**Key architectural decisions:**
- **App Router only** — no Pages Router. All layouts use RSC by default.
- **RTL/LTR split** — root layout is RTL (`fa-IR`). `/en` subtree wraps in LTR `<div>` via `EnglishShell`.
- **Security-first API** — every state-changing route verifies CSRF origin before processing.
- **Fail-closed defaults** — missing env secrets block requests in production.
- **SEO-first** — every route has canonical URL, locale-specific metadata, and Schema.org structured data.
- **AI context injection** — AI Mentor receives student performance data in every conversation.

## Installation

### Prerequisites

- Node.js 22 (see `.nvmrc`)
- PostgreSQL 15+
- npm 10+

### Local Setup

```bash
# 1. Clone the repository
git clone https://github.com/tecpey/Tecpey-Os.git
cd Tecpey-Os

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.local.example .env.local
# Edit .env.local and fill in all required values

# 4. Run the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to see the result.

### Required Environment Variables

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `TECPEY_SESSION_SECRET` | Session JWT secret (min 32 chars) |
| `TECPEY_ACADEMY_AUTH_SECRET` | Academy auth JWT secret (min 32 chars) |
| `CERTIFICATE_SIGNING_SECRET` | Certificate signing secret (min 32 chars) |
| `NEXT_PUBLIC_SITE_URL` | Full site URL (e.g. `https://tecpey.ir`) |
| `NEXT_PUBLIC_API_URL` | Exchange API URL |

See `.env.local.example` for the full list.

## Development

```bash
# Development server with hot reload
npm run dev

# Type checking
npx tsc --noEmit

# Linting (must produce 0 warnings)
npm run lint

# Build for production
npm run build

# Start production server
npm start
```

### Code Standards

- TypeScript strict mode — all new code must be type-safe
- ESLint — 0 errors, 0 warnings (enforced by CI)
- No `any` casts without explicit justification
- Server Components by default — use `"use client"` only when necessary
- CSRF guard on every state-changing API route
- No hardcoded secrets — all sensitive values via environment variables
- No comments unless the WHY is non-obvious

## Security

TecPey takes security seriously.

- All state-changing API routes are CSRF-protected
- JWT secrets use fail-closed defaults in production
- Admin sessions expire after 15 minutes
- Password minimum: 10 characters
- Anti-phishing education is built into the Academy onboarding flow
- Security headers enforced via Next.js middleware (X-Frame-Options, CSP, Referrer-Policy)
- No `X-Powered-By` header in production

To report a vulnerability, see [SECURITY.md](./SECURITY.md).

**Do not open public GitHub issues for security vulnerabilities.**
